# 港股新股推荐与 ROI 预测看板 — 智能体协同规范

## 项目简介
本项目通过多智能体协同管线，自动化采集港股 IPO 数据并进行量化分析，最终以 Streamlit 看板呈现推荐结果。

---

## 智能体清单与职责

### 🧠 Orchestrator Agent（总指挥）
- **职责**：编排所有子 Agent 的执行顺序，汇总数据为统一 JSON，写入 `st.session_state`
- **规则**：
  - 启动时一次性调度所有数据 Agent，结果缓存到 Session State
  - 后续组件切换不触发重新计算，全部基于内存读取
  - 仅在用户点击「刷新数据」时才重新执行完整管线

### 📡 IPO Scanner Agent（新股雷达）
- **职责**：追踪港交所最新招股动态
- **输出**：正在招股 / 即将上市的新股列表（名称、代码、日期、入场费）

### 💰 Fundamentals Agent（基本面分析师）
- **职责**：用大白话提炼公司主营业务与核心财务指标
- **输出**：业务描述、收入、利润、毛利率、赛道标签

### 🏛️ Cornerstone Agent（基石猎人）
- **职责**：提取基石投资者名单及认购比例
- **输出**：基石投资者列表 `[{name, amount, pct}]`、总占比、锁定期

### 📈 Sponsor Agent（保荐人评审官）
- **职责**：分析保荐人近两年的历史战绩
- **输出**：保荐项目数、首日破发率、平均首日涨幅

### 🔥 Sentiment Agent（市场温度计）
- **职责**：采集市场孖展认购倍数与散户讨论热度
- **输出**：孖展倍数估算、热度等级、舆情关键词

### 🎲 Quant Agent（量化策略师）
- **职责**：基于所有维度数据，计算 AI 推荐指数与 ROI 预测
- **输出**：综合评分(0-100)、ROI 区间、破发概率、行动建议
- **特殊规则**：
  - 必须引入**大盘调节系数 MarketBeta**，当恒生指数处于连跌趋势时下修 ROI
  - 评分模型权重：基本面 25% + 基石 20% + 保荐人 20% + 市场热度 20% + 定价 15%

## 全局规则

### 1. 时间戳过滤（Time-stamp Filtering）
> 所有子 Agent 在 `live` 模式下进行 Web Search 时，**必须**对搜索结果进行时间过滤（限定在最近 1-2 个月内），防止抓取到数年前同名公司的历史上市旧闻。

### 2. Session State 管理
> Orchestrator 汇总的完整 JSON 数据在启动时一次性载入 `st.session_state`。后续所有组件（详情面板、图表、ROI 预测）的渲染均基于 Session 内存读取，**禁止**在组件切换时重新调用 Agent。

### 3. 数据模式
- **Demo 模式（默认）**：使用 `data/sample_data.py` 中的静态模拟数据
- **Live 模式（可选）**：通过 Web Search 实时采集，结果缓存至 `data/cache/`

### 4. 代码规范
- 所有注释与文档使用**中文**
- 数据模型使用 **Pydantic v2**
- 前端使用 **Streamlit ≥ 1.35** + **Plotly**
- 异步函数统一使用 `async def run()` 接口
